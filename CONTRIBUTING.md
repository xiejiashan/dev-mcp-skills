# Contributing

Thank you for your interest in contributing! We welcome pull requests to improve existing skills, add new tools or examples, and enhance documentation.

## Workflow

1. Fork this repository and create a new branch for your change.
2. Follow the [SPEC](docs/spec.md) when adding or updating skills and tools.
3. Add or update tests/examples as appropriate.
4. Submit a pull request with a clear description of your change.

## Code Style

- Use meaningful commit messages.
- Keep changes focused; avoid large PRs.
- For Python code, follow PEP 8 where applicable.
- For YAML/JSON, maintain indentation and field ordering consistent with existing files.
